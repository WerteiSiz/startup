# Шаг 1. Гипотезы теста и критерии успеха

## 1.1 Сегмент, сценарий, ценность

| Параметр | Значение |
|----------|---------|
| Сегмент | Молодые люди 18–35 лет (студенты, молодые специалисты), активно посещающие кафе, мероприятия и развлекательные места |
| Сценарий | Пользователь хочет провести досуг (встреча с друзьями, свидание, отдых) → открывает сайт → ищет место со скидкой → выбирает и переходит к посещению |
| Ценностное предложение (UVP) | Найди куда сходить и сэкономь — все скидки на досуг в одном месте |
| Целевое действие | Пользователь находит подходящее место и сохраняет его (или переходит к использованию предложения) |

---

## 1.2 Список гипотез

| ID | Тип | Гипотеза | Критерий подтверждения | Критерий опровержения |
|----|-----|----------|------------------------|------------------------|
| H1 | Ценность | Пользователи считают скидки полезными и экономящими деньги | ≥ 70% говорят “это полезно” | < 50% |
| H2 | Сценарий | Пользователь быстро находит подходящее место через фильтры | ≥ 70% находят вариант ≤ 3 мин | > 50% не справляются |
| H3 | UX | Интерфейс сайта понятен без объяснений | ≤ 30% испытывают трудности | ≥ 50% жалоб |
| H4 | Retention | Пользователь готов вернуться для повторного поиска | ≥ 50% готовы вернуться | < 30% |
| H5 | Монетизация | Пользователь готов использовать платные функции (например, premium доступ) | ≥ 25% готовы платить | < 15% |

# Шаг 5. Сбор обратной связи

## 5.1 Респонденты (10 человек)

- 6 студентов (18–23 года)  
- 4 молодых специалиста (24–30 лет)  

---

## 5.2 Сценарий теста

1. Пользователь заходит на сайт  
2. Ищет место через фильтры  
3. Открывает карточку  
4. Добавляет в избранное  
5. Отвечает на вопросы  

---

## 5.3 Вопросы

- Что понравилось?  
- Было ли сложно найти место?  
- Использовал бы снова?  
- Заплатил бы за расширенные функции?
