## 2. Подводные камни и риски

### 2.1. Верификация студентов

| Риск | Описание | Митигация |
|------|----------|-----------|
| Подделка студенческих | Пользователи могут загружать фото чужих билетов | Использовать .edu почту или интеграцию с Госуслугами |
| Задержка верификации | Ручная проверка может занять дни, пользователи уйдут | Сделать автоматическую проверку по .edu почте |
| Юридические ограничения | Хранение персональных данных требует согласия | Добавить политику конфиденциальности и согласие |

### 2.2. Партнеры и скидки

| Риск | Описание | Митигация |
|------|----------|-----------|
| Неактуальные скидки | Партнеры забывают удалять истекшие скидки | Автоматическая деактивация по дате + уведомления партнерам |
| Злоупотребление | Один студент активирует скидку много раз | Лимит на пользователя + проверка по email |
| Партнеры не платят | Если модель с оплатой — сложно собирать деньги | Предоплата или интеграция с платежной системой |
| Недостаток партнеров | На старте может быть мало скидок | Ручной обзвон кафе + партнерские программы |

### 2.3. Технические риски

| Риск | Описание | Митигация |
|------|----------|-----------|
| Нагрузка на базу | Резкий рост пользователей | Кэширование (Redis), индексы в БД |
| Безопасность | Утечка паролей или данных | Хэширование паролей (bcrypt), JWT с коротким сроком |
| QR-коды | Генерация и валидация требует времени | Кэшировать QR, использовать простые алгоритмы |
| API rate limiting | Боты могут нагружать систему | Ограничить количество запросов с одного IP |

### 2.4. Бизнес-риски

| Риск | Описание | Митигация |
|------|----------|-----------|
| Конкуренты | Есть существующие агрегаторы скидок | Уникальное УТП: верификация студентов + AI |
| Низкая активность | Пользователи регистрируются, но не используют | Push-уведомления, геймификация (баллы) |
| Сезонность | Летом активность студентов падает | Специальные летние предложения |

### 2.5. Юридические риски

| Риск | Описание | Митигация |
|------|----------|-----------|
| Персональные данные | Хранение email, телефонов, фото студенческих | Соответствие 152-ФЗ, шифрование данных |
| Договоры с партнерами | Устные договоренности — риск неоплаты | Типовой договор, электронная подпись |
| Промокоды и скидки | Партнер может отказаться признавать скидку | Письменное соглашение + QR с подтверждением |

## 3. Схема базы данных

### 3.1. PostgreSQL (реляционная модель)

```sql
-- Пользователи
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    full_name VARCHAR(255),
    phone VARCHAR(20),
    is_verified BOOLEAN DEFAULT FALSE,
    verification_method VARCHAR(50), -- 'email', 'photo', 'gosuslugi'
    role VARCHAR(50) DEFAULT 'user', -- 'user', 'admin', 'partner_admin'
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Партнеры (кафе, кино, музеи)
CREATE TABLE partners (
    id SERIAL PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    description TEXT,
    category VARCHAR(100) NOT NULL, -- 'cinema', 'cafe', 'museum', 'food_delivery', etc.
    address TEXT,
    metro_station VARCHAR(100),
    latitude DECIMAL(10, 8),
    longitude DECIMAL(11, 8),
    logo_url TEXT,
    contact_person VARCHAR(255),
    contact_phone VARCHAR(20),
    contact_email VARCHAR(255),
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Скидки
CREATE TABLE discounts (
    id SERIAL PRIMARY KEY,
    partner_id INTEGER REFERENCES partners(id) ON DELETE CASCADE,
    title VARCHAR(255) NOT NULL,
    description TEXT,
    discount_type VARCHAR(50) NOT NULL, -- 'percentage', 'fixed_amount', 'promo_code'
    discount_value INTEGER NOT NULL, -- процент (20) или сумма (500)
    promo_code VARCHAR(100), -- если тип promo_code
    qr_code_url TEXT, -- если генерируем QR
    valid_from DATE NOT NULL,
    valid_to DATE NOT NULL,
    usage_limit INTEGER, -- общий лимит активаций (NULL = без лимита)
    per_user_limit INTEGER DEFAULT 1, -- сколько раз может активировать один пользователь
    used_count INTEGER DEFAULT 0,
    is_active BOOLEAN DEFAULT TRUE,
    terms TEXT, -- условия применения
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Избранное
CREATE TABLE favorites (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id) ON DELETE CASCADE,
    discount_id INTEGER REFERENCES discounts(id) ON DELETE CASCADE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(user_id, discount_id)
);

-- Активации скидок (история использования)
CREATE TABLE discount_uses (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id) ON DELETE CASCADE,
    discount_id INTEGER REFERENCES discounts(id) ON DELETE CASCADE,
    used_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(user_id, discount_id) -- защита от дублей (можно снять, если per_user_limit > 1)
);

-- Отзывы
CREATE TABLE reviews (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id) ON DELETE CASCADE,
    discount_id INTEGER REFERENCES discounts(id) ON DELETE CASCADE,
    rating INTEGER CHECK (rating >= 1 AND rating <= 5),
    comment TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Сессии (для JWT refresh токенов)
CREATE TABLE sessions (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id) ON DELETE CASCADE,
    refresh_token VARCHAR(512) NOT NULL,
    expires_at TIMESTAMP NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
